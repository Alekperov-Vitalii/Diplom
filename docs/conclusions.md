# ВИСНОВКИ

У процесі виконання курсової роботи було спроектовано та розроблено програмне забезпечення для веб-системи моніторингу та керування охолодженням GPU-кластера. Реалізована система забезпечує комплексний моніторинг стану восьми GPU, автоматичне адаптивне керування системою охолодження на основі каскадного алгоритму, візуалізацію даних у реальному часі та можливість ручного втручання в роботу системи. Система відповідає всім висунутим вимогам та забезпечує надійну та ефективну роботу системи охолодження GPU-кластера.

Отримані результати підтверджують відповідність розробленого програмного забезпечення завданню на виконання роботи та висунутим вимогам. Система реалізує всі основні функціональні вимоги: збір телеметричних даних від апаратного рівня (ESP32 або емулятора), збереження даних у часовісній базі даних InfluxDB, автоматичне керування вентиляторами на основі температур GPU та приміщення, візуалізацію поточного стану системи та історичних даних, ручне керування системою охолодження, систему алертів при критичних температурах. Нефункціональні вимоги також виконані: система забезпечує оновлення даних кожні 5 секунд, має адаптивний інтерфейс для різних пристроїв, реалізує валідацію даних та захист від небезпечних дій користувача.

Архітектурні рішення, прийняті при проектуванні системи, забезпечують масштабованість, надійність та підтримуваність програмного забезпечення. Використання багаторівневої клієнт-серверної архітектури з розділенням відповідальності між компонентами дозволяє незалежно масштабувати окремі рівні системи та спрощує тестування та підтримку коду. Застосування патерну Fog Computing забезпечує обробку даних ближче до джерела, що зменшує затримки та навантаження на мережу, що критично важливо для систем реального часу.

Технологічний стек, обраний для реалізації системи, забезпечує високу продуктивність та сучасність рішення. Використання Next.js для веб-інтерфейсу дозволяє застосовувати Server-Side Rendering та Static Site Generation для оптимізації продуктивності, React забезпечує реактивність інтерфейсу, TypeScript – типобезпеку коду. FastAPI для серверної частини забезпечує високу продуктивність завдяки асинхронній обробці запитів та автоматичній генерації документації API. InfluxDB як часовісна база даних оптимізована для роботи з часовими рядами та забезпечує ефективні запити за часовими діапазонами.

Проектування інтерфейсу користувача здійснено з урахуванням принципів сучасного UX/UI дизайну та специфіки систем моніторингу реального часу. Реалізована візуальна інфраструктура з ефективним використанням кольорового кодування забезпечує швидке сприйняття стану системи, компонентна архітектура інтерфейсу забезпечує модульність та переіспользуваність коду. Механізм автоматичного оновлення даних через polling забезпечує актуальність інформації без необхідності ручного оновлення сторінки, що критично важливо для систем моніторингу.

Система валідації та безпеки на сторінці ручного керування запобігає небезпечним діям користувача та забезпечує контроль за станом системи. Візуалізація даних через інтерактивні графіки дозволяє користувачу аналізувати тренди та виявляти закономірності в роботі системи охолодження, що сприяє оптимізації роботи системи та діагностиці проблем.

Можливість впровадження або застосування результатів роботи підтверджується повною функціональністю системи та її готовністю до використання в реальних умовах. Система може бути розгорнута на виділених серверах або в хмарному середовищі, підтримує різні варіанти розгортання та може бути легко адаптована для різних конфігурацій GPU-кластерів. Емулятор апаратного рівня дозволяє тестувати систему без необхідності реального апаратного обладнання, що спрощує розробку та діагностику.

У подальшому проект може бути розширений додаванням нових функціональних можливостей та покращенням існуючих компонентів. Перспективними напрямками розвитку є інтеграція машинного навчання для прогнозування температур та оптимізації алгоритму керування охолодженням, додавання системи сповіщень через електронну пошту або SMS при критичних ситуаціях, реалізація WebSocket для real-time оновлення даних замість polling, додавання системи автентифікації та авторизації користувачів для багатокористувацького режиму, розширення аналітичних можливостей з додаванням звітів та експорту даних, інтеграція з системами моніторингу інфраструктури (наприклад, Prometheus, Grafana), додавання підтримки множинних GPU-кластерів з централізованим управлінням, реалізація мобільного додатку для моніторингу системи з мобільних пристроїв. Технічні покращення можуть включати оптимізацію продуктивності через кешування даних, додавання резервування компонентів для забезпечення високої доступності, реалізацію автоматичного масштабування компонентів залежно від навантаження, покращення системи логування та моніторингу для діагностики проблем. Архітектурні покращення можуть включати перехід до мікросервісної архітектури для кращої масштабованості, додавання message queue для асинхронної обробки даних, реалізацію event-driven архітектури для реакції на події в реальному часі. Всі ці напрямки розвитку забезпечать подальше вдосконалення системи та розширення її функціональності для задоволення нових потреб користувачів.
